Jury Q&A — Keep answers under 30 seconds

How is this different from SonarQube/Snyk?Dev answers They do static analysis only. We also run the app with an AI agent that clicks, types, and tests like a real user. And we learn from every scan.
Does the AI simulation actually work?Arun answers Yes — Claude Sonnet 4.6 via Playwright MCP gets 21 browser tools. It reads the accessibility tree, types XSS payloads, and confirmed a vulnerability by triggering alert() in the browser. Happy to run it live.
What about false positives?Arun answers Critical findings like leaked API keys are high-confidence regex matches. For XSS, we compare innerHTML vs textContent — only flag if the payload is actually unescaped. Simulation findings include the reproduction path.
Will this scale?Arun answers Scanners run in parallel, under 5 seconds for most repos. For scale, we'd add a worker queue and hosted browser pool.
Why not just use ESLint?Dev answers ESLint catches style issues. We catch leaked Stripe keys, hallucinated npm packages that don't exist, PII exposure, and runtime crashes that no linter can find.
What's the tech stack?Arun answers React, Node, Express, Playwright MCP, Claude Sonnet 4.6 via Anthropic API, SQLite, OSV.dev for CVE data.
What's the knowledge base?Dev answers Every scan stores failure patterns with frequency. After 34 scans we know the top 50 mistakes AI makes. Future scans use this data to prioritize testing.
How does the GitHub Action work?Arun answers It clones our scanner repo, runs 4 scanners, prints score. Below 40 = CI fails. One YAML file, every commit protected.
What's the social impact?Dev answers AI-generated code is everywhere — startups, students, enterprises. VibeCheck protects users from shipping code that leaks their data or has security holes.
Don't know the answer?Anyone "That's a great question — we haven't explored that yet, but our hypothesis is..." Never bluff. Judges respect honesty.

If Things Go Wrong

Live demo crashes
Abhijay plays backup video. Dev says "Let me show you the recording." Never say "it was working 5 minutes ago."
Scan takes too long
Talk about architecture while it loads. "Under the hood, 4 scanners run in parallel with Playwright MCP..."
AI Simulation fails (rate limit)
Skip it. "The static analysis found 12 issues. Let me show you the code heatmap instead."
WiFi dies
Scan local path /tmp/test-vibe-app instead — works completely offline.
Judges ask to see code
Open GitHub: github.com/Arun07AK/vibecheck. Show commit history — built today.
Don't know the answer
"Great question. We haven't explored that yet, but here's our hypothesis..." Judges respect honesty over bluffing.
